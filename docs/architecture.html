Factory Inventory Management

Full-stack system architecture — Vue 3, FastAPI, in-memory data

System Architecture

Tech Stack

Frontend

  • Vue 3 + Composition API
  • Vue Router 4
  • Vite (dev + build)
  • Axios (HTTP client)
  • Custom SVG charts
  • CSS Grid + Flexbox
  • i18n (EN / JA)

Backend

  • Python 3 + FastAPI
  • Uvicorn (ASGI)
  • Pydantic v2 (validation)
  • CORS middleware
  • 18 REST endpoints

Data

  • 7 JSON files (~395 KB)
  • In-memory at runtime
  • No database (demo)
  • Loaded on server start

Tooling

  • uv (Python packaging)
  • npm (frontend deps)
  • Makefile (orchestration)
  • pytest + httpx (tests)
  • 55 passing tests

Data Flow

1
User Action
Selects filters or navigates to a view in the Vue SPA
2
API Client
api.js builds URL with query params via Axios
3
FastAPI Route
Receives request, extracts filter parameters
4
Filter Engine
Applies warehouse, category, status, month filters sequentially
5
Pydantic
Validates and serializes response models
6
Render
Vue computed properties transform data into charts and tables

Application Features

Dashboard

5 KPI cards (turnover, fulfillment, revenue), donut/bar/pie charts for order health and inventory value, shortage alerts, top products.

Inventory Management

Searchable table of 20 SKUs across 3 warehouses and 5 categories. Stock status badges, detail modals, real-time filtering.

Order Tracking

144+ orders with 4 statuses (Delivered, Shipped, Processing, Backordered). Sortable table with expandable line items.

Spending Analytics

Revenue vs costs by month, category breakdown (Procurement, Operational, Labor, Overhead), transaction history.

Demand Forecasting

Current vs forecasted values with trend indicators (Increasing, Stable, Decreasing) organized by warehouse.

Reports

Quarterly metrics (orders, revenue, fulfillment rate) and monthly trends with year-over-year comparisons.

Internationalization

Full English/Japanese support with automatic currency conversion (USD/JPY at 150:1 rate). LocalStorage persistence.

Filter System

4 independent filters (Time Period, Warehouse, Category, Status) applied globally across all views via shared composable.
